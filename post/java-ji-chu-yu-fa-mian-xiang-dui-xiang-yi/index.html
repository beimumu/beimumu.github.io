Java基础语法_面向对象（一）
发表于：2020-10-05 | 分类： Java
字数统计: 2.7k | 阅读时长: 10分钟 | 阅读量：

1.面向过程 & 面向对象

  1. 面向过程（结构化程序设计）：
    • 程序=算法+数据结构
    • 算法第一，数据结构第二
    • 面向过程的编程需要我们对过程的每一步进行严格控制，并且每个过程之间联系非常紧密。
  2. 面向对象：
    • 程序=对象+对象+对象+…
    • 数据第一，算法第二
    • 关键：让每一个对象负责执行一组相关任务
    • 面向对象的编程我们只需要把任务拆分为几个独立的小任务，然后由每个对象来完成这些小任务，并且这些人物之间关系没那么紧密。
  3. 举例：青椒炒肉的做法逻辑。

2.类和对象

  1. 类：类是模板，类是蓝图，是一组类似对象的共同抽象定义。

    例：狗有四条腿，猫有四条腿，老虎有四条腿，狮子也有四条腿。他们都有犬牙、可以捕猎以及吃肉等。

    总结：提取个体中他们共同的特性，定义一个新的名称叫“食肉动物”。我们把新的物种成为有一个新的类。先从个体中总结出一个共有的类（共同属性的集合），在从类中实例出具体的对象。

  2. 对象：对象由类产生出来的，具有上述共同特性的一个单独的个体。对象是类的一个实例化的具体表现。

  3. 在Java中：先有类，再有对象 。

  4. 类可以当作一个有共同特点的集合，而对象就相当于在名词前加了一个定语。

  5. 类存在的目的就是实例化对象。

3.类的创建

  1. 是定义类名
  2. 编写属性代表类的静态特性。比如：姓名、年龄、学号。
  3. 编写方法代表类的动态行为（操作）。比如：看书、打游戏、学习。

4.对象的创建

以Student类为例

Student stu = new Student();
//类名 变量名 = new 类名();

  1. 在栈中声明引用变量：Student stu

  2. 在堆中开辟一个内存空间用来存储变量值。执行new Student();

    堆中占多大内存取决于属性是什么数据类型。

  3. 使第一步栈中的引用变量指向堆中的对象。

  4. 导航符.可以修改属性的值和调用方法(通过对象发送消息的的功能体现)。

特点：这样我们就可以通过stu这个变量来找到堆内存中的这个对象，也可以通过stu改变堆内存中属性的值。

注：每创建一个对象，在堆中内存就会开辟一个内存空间，而在方法区的内存只占一份。在方法区中，哪个对象调用方法，这个方法就有调用他的对象的属性。举例：磨坊磨小麦。

5.方法的定义

访问修饰符 返回值类型 方法名（参数列表）{
    //方法体
}
public void exam(){
    System.out.println(name+"在考试");
}

  1. 访问修饰符：

    • 控制当前方法对其他类或其他方法的访问权限。

    • public、protected、默认的、private。

      访问权限由低→高

  2. 返回值类型：

    • 返回值类型是指方法执行结束以后，需要向外返回的数据类型。可以是基本数据类型，也乐意是引用数据类型。

    • 把方法当作一个模块(比如”黑匣子“)，完成某个特定的功能后，返回一个处理结果。

  • 如果不需要外部提供一个返回值，需要使用关键字void。

  • 如果方法有返回值，方法体中必须使用关键字return返回该值，返回值类型为该方法所定义的返回值类型。

  • 调用此方法时，可以声明一个变量来接收此方法的返回结果。

    在返回值类型为String的方法下会报错（丢失返回值语句），即需要我们写一个返回String类型的结果。代码：return "这是String类型的返回结果";

    stu.aaa();
System.out.println(stu.aaa());

    如上图所示：调用aaa()方法，不会返回String类型的结果。

    而输出打印aaa()方法的话，不仅会有aaa()方法的输出语句，还会有其String类型的返回结果。

    String res = stu.aaa();
System.out.println(res);

    如上图所示：声明一个String类型的res变量，用来接收aaa()方法的返回结果。

    注：用户是否能接受到返回信息取决于是否需要给用户返回一个结果，而是否需要给用户返回一个结果取决于是否需要设置一个res(”结果“的变量)。

练习：创建一个JudgeAge类，根据年龄返回不同的字符串。

创建JudgeAge类：

public class JudgeAge {
    public String suanMing(){
        Scanner sc = new Scanner(System.in);
        System.out.println("请输入你的年龄：");
        int age = sc.nextInt();
        String res = "";
        if (age < 18){
            res = "男儿志在四方！";
        }else if (age >=18 && age <40){
            res = "再不疯狂，我们就老了！";
        }else{
            res = "谁还不是个宝宝呢！";
        }
        return res;
    }
}

创建TestJudeAge类：

public class TestJudeAge {
    public static void main(String[] args) {
        JudgeAge jud = new JudgeAge();
        jud.suanMing();
    }
}

如果是这样，则没有返回的字符串，因为我们没有设置返回结果的变量，只是调用了suanMing()方法，并没有选择去接收我们return的结果。

String res = jud.suanMing();
System.out.println(res);

设置一个返回结果的变量，输出就没毛病了。

jud.suanMing();
System.out.println(jud.suanMing());

显然，这种输出方式使同一个语句出现两次重复了。

总结：可以想到上网购物下单成功后给你提供一个“下单成功”的返回结果。

  1. 方法名：

    符合驼峰规则。字母、下划线、$、数字，数字不能开头。

  2. 参数列表：

    • 在方法的定义中，括号里的出现的参数(形式参数）。形参个数不固定，也可以没有。只能在此方法内部使用。

    • 在调用方法的时候们必须提供参数的个数，顺序，数据类型完全一致的参数。这时的参数叫实参(实际参数)。

    练习：创建计算器类完成简单计算。

    • 创建Calc类：

      public class Calc {

    //加法运算
    public double add(double a,double b){
        double res = a+b;
        return res;
    }

    //减法运算
    public double minus(double x,double y){
        double res = x-y;
        return res;
    }

    //乘法运算
    public double mcl(double c,double d){
        double res = c*d;
        return res;
    }

    //除法运算
    public double divide(double m,double n){
        double res = m/n;
        return res;
    }
}

    • 创建测试类：

      public class TestCalc {
    public static void main(String[] args) {
        Calc calc = new Calc();
        Scanner sc = new Scanner(System.in);
        System.out.print("请输入第一个数字：");
        double a =sc.nextInt();
        System.out.print("请输入第二个数字：");
        double b =sc.nextInt();
        System.out.print("请选择运算符号:");
        String str =sc.next();
        //也可以用switch语句
        if (str.equals("+"))
        {
            double res = calc.add(a, b);
            System.out.println(a+"+"+b+"="+res);
        }
        if (str.equals("-"))
        {
            double res = calc.minus(a,b);
            System.out.println(a+"-"+b+"="+res);
        }
        if (str.equals("*"))
        {
            double res = calc.mcl(a, b);
            System.out.println(a+"*"+b+"="+res);
        }
        if (str.equals("/"))
        {
            double res = calc.divide(a, b);
            System.out.println(a+"/"+b+"="+res);
        }
    }
}

  3. 方法签名：

    //方法名+参数列表叫做方法签名
add(double a,double b);    
double minus(double x,double y);
double divide(double m,double n); 
//同名不同参
add(double a,double b);
add();
add(double a,double b,double c);

    在同一个类中，不允许出现方法签名相同的方法。

  4. 方法的重载(overload)：方法名相同，参数列表不同的一组方法。（即：**同一个类中**，同名不同参），同一个类中的方法就可以同时存在。

    典型应用：构造方法重载，System.out.printlln();

  5. 变量的作用域：

    • 全局变量：类中在方法之外定义的变量。全局变量可以在当前类或其它类中被使用。
    • 局部变量：在方法内部声明的变量，只能在方法内部使用。形参也属于局部变量。 注意：局部变量未初始化不能使用。

    可以理解为自己的大名和小名。

  6. 值传递

    • 在Java中，方法调用，如果参数为基本数据类型，那么实参传递的实际的数值。

      //ValueDeliver类
public void updateValue(int num){
        num = num+5;
    System.out.println("updateValue方法："+num);
}

//TestValueDeliver类
ValueDeliver va = new ValueDeliver();
int a = 5;
va.updateValue(a);//输出的是此方法中的结果：updateValue方法：10
System.out.println(a);// a的值为5

    • 在Java中，方法调用，如果参数为引用数据类型，那么实参传递的是引用值（内存地址）。

      //Account类
public class Account {
    int cou_num = 1001;//账户编号
    String user_name = "test";//用户名
    String password = "test";//密码
    double yu_money = 1234;//余额
}

//ValueDeliver类
public void updateAccount(Account account){
        account.cou_num = 2002;
        account.user_name = "adm";
        account.password = "123";
        account.yu_money = 123;
    }

//TestValueDeliver类
		ValueDeliver va = new ValueDeliver();
		Account account = new Account();
        //调用Account类的变量，原样输出，值没变。
        System.out.println(account.cou_num); //1001
        System.out.println(account.user_name); //"test"
        System.out.println(account.password); //"test"
        System.out.println(account.yu_money); //1234
        //不管怎么传递，account对象在内存中只有一份，方法内操作对象，（内存空间相当于一个盒子）
        //首先，需要在内存中找到这个对象然后操作。
        va.updateAccount(account);
        //调用了updateAccount方法，值发生改变。
        System.out.println(account.cou_num); //2002
        System.out.println(account.user_name); //"adm"
        System.out.println(account.password); //"123"
        System.out.println(account.yu_money); //123

  7. Java对象的内存管理机制

    首先回顾下Java内存的模型：

    1. 使用new创建对象，则会在堆内存中开辟新的空间和初始化。
    2. 方法执行完毕返回，栈内存自动释放，局部变量销毁（此时堆中的对象还存在，只是没有任何变量来指向这个对象）。
    3. 如果堆内存中的对象没有被变量引用，就会变成垃圾，此时由垃圾回收器回收，释放所占的堆内存。
    4. 被回收的对象：
      • 不再被任何变量引用的对象
      • 引用变量自动放弃
      • 认为将引用变量置为null
    5. System.gc();手动调用垃圾回收机制。但还有个问题，此方法执行之后，何时扫描，合适会回收，用户无法干预，他有自己的算法。
