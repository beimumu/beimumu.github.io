21.异常

1.异常机制

  • 异常的本质：当程序出现异常，程序可以安全退出，处理完后继续执行的机制。

  • java异常也是对象，所以采用面向对象方式处理异常：**try-catch()**的过程：

    //1.抛出异常
try{
    //将异常生成对象，并传给JVM
}
//2.捕获异常
catch(Exception e){
    //JVM寻找对应的异常解决方法来解除异常
    e.printStackTrance();
}

  • Throwable是所有异常的父类。Error类：人为解决不了的问题，一般重启大法。Exception：则是我们所需要学习和将来打交道的异常。

  • RuntimeException：运行时异常，由于编译器检查不出来的异常也叫（UncheckeException）。

    • 解决异常的方法之一就是：从逻辑上解决，手动加判断解决。

      • 空指针异常

        public class Demo02 {
    public static void main(String[] args) {

        String str = null;
        if (str != null){
            System.out.println(str.charAt(0));
        }

    }

}

2.异常处理

  • 捕获异常（try-catch-finally）

    • try

      用来执行程序，如果发现异常，会生成异常对象交给catch来解决。

    • catch

      处理可能产生的不同类型的异常对象。

      语句中可以有多个catch。

      若存在关系，则子类在前，父类在后；

      若没有关系，前后位置无所谓。

    • finally

      最多只能有一个，异常处理提供的一个出口，所指定的代码都要被执行。

      作用：释放资源。

    • try中有return时

      public class Demo06 {

    public static void main(String[] args) {
        int t = test();
        System.out.println(t);
    }

    public static int test(){
        int num = 5;
        try {
            return num;
        }catch (Exception e){
            e.printStackTrace();
        }finally {
            num++;
            System.out.println("num的值："+num);
        }

        System.out.println("11111");
        return num;
    }
}

      如果try语句中有return，会发生的异常现象：

      1、try中若没出现异常，则不管有没有finally，程序都不会执行try-catch以外的代码。此时return语句会执行，但会把return的结果先缓存起来，然后去执行finally，等finally执行完后，再回过头将缓存的return值取出来，然后真正的return(取出来的值)。

      2、 try中若出现异常，程序则会按照顺序进行执行完毕。return出最后执行的值。

      如果try中有多个语句，有异常时则跳过剩余的语句不执行，等catch解决后进入finally，再继续按顺序执行的剩下的语句。

    • 常用方法：（均继承自Throwable类）

      常用方法 说明
      e.toString(); 显示异常的类名和产生异常的原因
      e.getMessage(); 只显示原因，不显示类名
      e.printStackTrace(); 用来跟踪异常事件发生时堆栈的内容

    • 文件流例子

      //核心代码
FileReader reader = new FileReader("e:/demo.txt");

char c = (char) reader.read();
char c2 = (char) reader.read();
char c3 = (char) reader.read();

System.out.println(""+c+c2+c3);

reader.close();

      对核心代码进行异常处理：

      public class Test01 {
    public static void main(String[] args) {
        //外部声明
        FileReader reader = null;
        try {
            //try里初始化
            reader = new FileReader("e:/demo.txt");//FileNotFoundException -> OException

            char c = (char) reader.read();//IOException
            char c2 = (char) reader.read();
            char c3 = (char) reader.read();
            System.out.println(""+c+c2+c3);

        }catch (FileNotFoundException e){
            e.printStackTrace();
        }catch (IOException e){
            e.printStackTrace();
        }finally {
            try {
                if (reader != null){
                    reader.close();//IOException
                }
            }catch (IOException e){
                e.printStackTrace();
            }
        }
        
    }
}

      try相当于一个代码块，即封闭状态，在处理关闭IO流时，外面访问不到，所以需要在外面声明，只在try内部初始化。

  • 声明异常（throws）

    • 只是推迟解决异常，不是说我不处理。我有异常，但是我不抛出，即我不使用catch语句，我在方法前声明异常（throws），最后谁调用我，谁来catch。

    • 声明异常后，catch内容可以不写，但finally必须存在，起到一个关闭资源的作用。

    • 文件流例子

      public class Test03  {
    public static void main(String[] args) throws Exception {
        
        FileReader reader = null;
        reader = new FileReader("e:/demo.txt");

        char c = (char) reader.read();
        char c2 = (char) reader.read();
        char c3 = (char) reader.read();

        System.out.println(""+c+c2+c3);
    }
}

      调用声明解决异常：

      public class Test04 {
    public static void main(String[] args) /*throws Exception*/ {
        try {
            readfile("d:/demo.txt");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    //声明异常方法
    public static void readfile(String path) throws Exception{
        FileReader reader = null;
        try {

            reader = new FileReader(path);
            char c = (char) reader.read();
            char c2 = (char) reader.read();
            char c3 = (char) reader.read();
            System.out.println(""+c+c2+c3);

        }finally {
            try {
                if (reader != null){
                    reader.close();
                }
            }catch (IOException e){
                e.printStackTrace();
            }
        }
    }
}

  • try-with-resource 自动关闭Closable接口资源

    • JDK7后提供的一个语法糖，简化了try-catch，方便我们使用。JVM编译时还是进行的是try-catch-finally。

    • 它可以自动关闭实现了AutoClosable接口的类，实现类需要实现close()方法。

    • 语法格式

      try(需要打开的资源1;[需要打开的资源2]...){
    //语句;
}catch(Exception e){
    e.printStackTrace();
}

    • 文件流例子

      public class Test05 {
    public static void main(String[] args) {

        try (FileReader reader = new FileReader("e:/demo.txt");){

            char c = (char) reader.read();//IOException
            char c2 = (char) reader.read();
            char c3 = (char) reader.read();
            System.out.println(""+c+c2+c3);

        }catch (Exception e){
            e.printStackTrace();
        }
    }
}

  • 自定义异常

    • 在程序中，可能会遇到JDK提供的任何标准类都无法充分描述清楚我们想要表达的问题，此时，我们可以创建自己的异常类，即自定义类。

    • 只需从Exception类或其他子类派生一个类即可。

    • 如果继承Exception类，则为受检查异常，必须对其进行处理，如果不想处理，就让自定义异常类继承RuntimeException类。

    • 习惯上，自定义应包含2个构造器：一个默认构造器（无参），另一个带有详细信息的构造器（有参：信息存储在Message）。

    • 语法格式

      class A_Exception extends Exception {
    //默认构造器（无参）
    public A_Exception(){
        
    }
    //带有详细信息的构造器
    public A_Exception(String message){
        super(message);
    }
}

3.利用百度等解决异常

正常学习和开发中，我们经常会遇到各种异常。大家在遇到异常时，需要遵循下面四步来解决：

  1. 细心查看异常信息，确定异常种类和相关Java代码行号
  2. 确定上下文相关的一些关键词信息（疑难问题，需要）
    • 寻找问题本身的关键词（名词）
    • 寻找问题上下文的关键词（名词）
    • 尽量细致的描述问题，开始搜索
    • 如果没找到，慢慢减少关键词，扩大搜索范围。
  3. 拷贝异常信息到百度，查看相关帖子，寻找解决思路
  4. 前两步无法搞定，再问同学或同事
  5. 前三步无法搞定，请示领导。
