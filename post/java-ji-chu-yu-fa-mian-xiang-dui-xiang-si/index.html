9.多态

1.多态的概念

  • 多态：一种行为，多种状态。

    行为：通过方法的重写或者方法的调用。

  • 实现多态的前提

    1. 存在继承关系
    2. 子类可以重写父类
    3. 父类引用的变量指向子类的对象

  • 以Person w = new Worker();举例：

    其中，w是父类Person类的引用变量，Worker是子类的对象。

    1、在Worker类中，有特有的变量salary，方法work(); 而在Person类中没有，所以Person类在调用时是无法访问的。

    2、在Worker类重写了Person类中的sleep();和toString();后，Worker类调用会优先于Person类的重写方法。

  • 模型图示例：

  • 多态的现象特点

    1. 父类引用变量后，子类的特有的属性和方法会丢失，无法访问，父类的不受影响。
    2. 在子类重写父类的方法后，会先调用子类的重写过的方法。若子类没有进行重写，则会调用父类的。

2.对象向上、下转型

编译看左，执行看右，Person p = new Student为例：

右边为实际类型，运行时认右边。

左边为声明的变量类型，编译器只认左边。

  • 向上转型：

    • 继承图上来看，子类转向父类。

  • 类型上来看，较专用类型转向较普通类型。

    • 安全角度来看，向上转型是安全的，属于自动类型转换，在程序中不会报错，但是在转型过程中，会丢失子类的特有的属性和方法。这也是多态的体现。

    注：丢失子类方法：实际过程是编译过程中未识别子类方法。

    Student s = new Student("张三","男","1001","三年二班");
Person p = s;
//等价于
Person p = new Student("张三","男","1001","三年二班");
----------------------------------------------------
Worker w = new Worker("李四","男","1002",1000);
Person p = w;
//等价于
Person p = new Worker("李四","男","1002",1000);

  • 向下转型：

    • 首先，父类转向子类，范围大的转向范围小的，所以需要强制转换。

      //会报错
Person p = new Person("张三","男","1001");
Worker w = (Worker)p;

//强制转换我自己，是不会报错的。
Person p = new Student("张三","男","1001","三年二班");
Student s = (Student)p;

    • 其次，由于向下转型的强制转换，所以再转换过程中未知因素太多，存在不安全性。

      Worker类型转成Person类型（披着Person类的Worker类，其本质还是Worker类型）再转换成Student类型，那能说Worker是Student类型？当然不合适，所以存在不安全性。

    • 在对象做下转型之前，没有其他有关对象类型信息时务必使用instanceOf关键字来进行判断，以此来避免classCastException异常。

      Person p = new Student();
System.out.println(p instanceof Person);//true
System.out.println(p instanceof Student);//true
System.out.println(p instanceof Worker);//false


//p instanceof Student,判断p是否是student类或者是student类子类的一个实例对象
Person p = new Student();
if (p instanceof Student){
    Student student = (Student)p;
    student.exam();
    System.out.println("Student类型");
}else if(p instanceof Worker){
    Worker worker = (Worker)p;
    worker.work();
    System.out.println("Worker类型");
}

3.静态绑定和动态绑定

  • 静态绑定

    • 编译期间方法绑定，又叫前期绑定。

    • 将方法和未来即将出现的对象进行绑定。使外界无法用对象进行调用。

    • 在Java中，“private，final，static，构造方法”都属于静态绑定

      private 私有类型，只允许本类访问
      final 不能重写，也就没有多态
      static 通过类名直接调用
      构造方法 通过JVM调用

    • 举例：Person.sleep();

  • 动态绑定

    • 只有在对象创建完成后，程序运行时才知道调用哪一个方法。

    • 根据变量实际引用的对象类型来决定调用哪一个方法。

    • 举例

      Person p = new Person();
p.sleep();

4.里氏替换原则

  • 里氏替换原则(Liskov Substitution Principle, LSP)：所有引用父类的地方必须能使用其子类的对象。

    从这个概念可以看出这个原则是面向对象多态的一种具体实践。通俗来讲 “老爸能干的事情，儿子都能干”， 因为儿子继承了老爸的基因。 反过来讲就不对了，时代在变化，新一代虽然继承了老一代的优良传统，但是在时代的影响下，新一代有了一些新的特性，老一代可能就不具备了，比如现在的年轻人会打游戏，但是他爸不一定会。老爸会骑自行车，换成儿子也能骑。

    同样的里氏替换原则告诉我们，在软件中将一个基类对象替换成它的子类对象，程序将不会产生任何错误和异常，反过来则不成立，如果一个软件实体使用的是一个子类对象的话，那么它不一定能够使用父类对象。

  • 典型应用：

    用父类作为形参，再传实参时，父类或者子类创建的实例对象依然能调用。

    打印机、动物叫。

  • 拓展

    由于使用父类对象的地方都可以使用子类对象，因此在程序中尽量使用父类类型来对对象进行定义，而在运行时再确定其子类类型，用子类对象来替换父类对象。通常我们会使用接口或者抽象方法定义基类，然后子类中实现父类的方法，并在运行时通过各种手段进行类型选择调用（比如反射）。

    在使用里氏替换原则时需要注意如下几个问题：

    (1)子类的所有方法必须在父类中声明，或子类必须实现父类中声明的所有方法。根据里氏替换原则，为了保证系统的扩展性，在程序中通常使用父类来进行定义，如果一个方法只存在子类中，在父类中不提供相应的声明，则无法在以父类定义的对象中使用该方法。

    (2) 我们在运用里氏替换原则时，尽量把父类设计为抽象类或者接口，让子类继承父类或实现父接口，并实现在父类中声明的方法，运行时，子类实例替换父类实例，我们可以很方便地扩展系统的功能，同时无须修改原有子类的代码，增加新的功能可以通过增加一个新的子类来实现。里氏替换原则是开闭原则的具体实现手段之一。这也就是我们应该更多的依赖抽象，尽量少的依赖实现细节。

5.练习

设计一个宠物类，作为父类，衍生出Dog，cat，tiger，lion等子类。设计一个宠物店，宠物店可以给动物洗澡。
原始方式：宠物店添加多个方法，每个方法负责给一种动物洗澡。
多态应用：宠物店只需要提供一个洗澡的方法，想给什么宠物洗澡，就自己new一个宠物传过来，然后宠物店中使用父类类型来接收参数。

public class Pet {
    private String name;
    private int id;
    private String sex;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public String getSex() {
        return sex;
    }

    public void setSex(String sex) {
        this.sex = sex;
    }
}
---------------------------------------------------------------
public class Dog extends Pet {

}
public class Cat extends Pet {

}
public class Pig extends Pet {

}
---------------------------------------------------------------
public class PetHouse {
    public void washPet(Pet pet){
        //如果给Pet传了属性，那么在输出时就要带上需要的属性的封装方法。
        System.out.println("给"+pet.getName()+"洗澡");
    }
}

public class TestPet {
    public static void main(String[] args) {
        PetHouse ph = new PetHouse();

        Dog p1 = new Dog();
        p1.setName("小黄");
        ph.washPet(p1);

        Cat p2 = new Cat();
        p2.setName("小花");
        ph.washPet(p2);
    }
}
