2.4 Set接口

2.4.1 HashSet容器类

  • HashSet是一个无序且不重复的集合，相当于高中数学中的集合。
  • 线程不安全
  • 允许有null值
  • 底层逻辑是“数组加单链表”的HashMap存储形式，只针对key（源码中会讲到，HashSet中存放的值都会存放到HashMap的key中，所以map中的key具备无序性和不可重复性）。
  • 增删查的效率很高
2.4.1.1 Hash算法原理
  • 又称散列算法，先取模运算，之后将计算得到的数据进行散列存储，目的是将计算到的数据越散越好。
  • 举例：图书馆交身份证。
2.4.1.2 HashSet的使用
public class HashSetDemo01 {
    public static void main(String[] args) {

        Set<String> set = new HashSet<>();

        //添加元素
        set.add("a");
        set.add("b2");
        set.add("c3");
        set.add("d");
        System.out.println(set); //[b2, c3, a, d]
        
        //删除指定元素
        boolean flag = set.remove("c3");
        System.out.println(flag);

        //获取元素，因为set没有索引，所以不能使用get(int index)方法
        //增强for循环
        for (String str : set) {
            System.out.println(str);
        }
        //Iterator迭代器循环
        Iterator<String> it = set.iterator();
        while (it.hasNext()){
            String next = it.next();
            System.out.println(next);
        }


        //是否包含某元素

        boolean isContains = set.contains("a");
        System.out.println(isContains);

        System.out.println(set); //[b2, c3, a, d]
        
    }
}

总结：List和Set的区别

都继承Collection接口，所以可以调用Collection接口下的所有方法；

但是在List中还有自己的一些抽象方法。其中，这些独有的抽象方法完全是为了List的特性（有序且重重）所打造的。有序即有索引，所以这些特有的抽象方法依据索引所创造。

2.4.1.3 HashSet的存储特征

HashSet的无序性

  • 在HashSet中底层使用HashMap存储元素的。HashMap底层使用“数组+单链表”实现元素的存储。

  • 元素在数组（容量默认是16）存放时，并不是有序存放也不是随机存放，而是对元素的哈希值进行运算（进行hashCode()方法）来决定元素在数组中的位置。

HashSet的不可重复性

当两个元素的哈希值进行运算后，在数组位置相同时，会调用元素的equals()方法进行判断：

  • 如果元素相同，就会发生覆盖。
  • 如果元素不相同，就会在当前数组的格子中使用单链表（放入到一个Node类，将这个结点对象再放入到数组当中）保存该元素。
2.4.1.4 HashSet存储自定义对象
public class User {

    private String name;
    private int age;

    public User() {
    }

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "User{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;

        User user = (User) o;

        if (age != user.age) return false;
        return name != null ? name.equals(user.name) : user.name == null;
    }

    @Override
    public int hashCode() {
        int result = name != null ? name.hashCode() : 0;
        result = 31 * result + age;
        return result;
    }
}
public class HashSetDemo02 {
    public static void main(String[] args) {

        HashSet<User> set = new HashSet<>();

        User u = new User("张三",18);
        User u1 = new User("张三",18);

        //存入set中
        set.add(u);
        set.add(u1);

        //查看哈希值
        System.out.println(u.hashCode());//460141958
        System.out.println(u1.hashCode());//1163157884

        //遍历set
        /*
        * 由于没有哈希判断，两个哈希值不一样，所以两个对象都存在。
        *
        * 重写了equals方法和hashCode方法后，两个哈希值（24021577）一样，为同一个对象。
        * */
        for(User user : set){
            System.out.println(user);
        }
    }
}
2.4.1.5 HashSet源码分析
private transient HashMap<E,Object> map;

// Dummy value to associate with an Object in the backing Map
private static final Object PRESENT = new Object();

/**
 * Constructs a new, empty set; the backing <tt>HashMap</tt> instance has
 * default initial capacity (16) and load factor (0.75).
 */
public HashSet() {
    map = new HashMap<>();
}

/**
 * Adds the specified element to this set if it is not already present.
 * More formally, adds the specified element <tt>e</tt> to this set if
 * this set contains no element <tt>e2</tt> such that
 * <tt>(e==null&nbsp;?&nbsp;e2==null&nbsp;:&nbsp;e.equals(e2))</tt>.
 * If this set already contains the element, the call leaves the set
 * unchanged and returns <tt>false</tt>.
 *
 * @param e element to be added to this set
 * @return <tt>true</tt> if this set did not already contain the specified
 * element
 */
public boolean add(E e) {
    return map.put(e, PRESENT)==null;
}

2.4.2 TreeSet容器类

  • TreeSet是一个可以对元素进行排序的容器。
  • 底层使用了TreeMap来实现的，通过key来存储Set的元素。
  • TreeSet内部需要对存储的元素进行排序，所以，我们需要给定排序规则。
    1. 通过元素自身实现比较规则（比如String类和Integer类实现了Comparable接口的CompareTo方法）。
    2. 通过外部比较器指定比较规则（创建一个比较器，实现Comparator接口中的compare方法）。
2.4.2.1 TreeSet的使用

与HashSet用法相同，不同的是会对元素自身进行一个排序。

2.4.2.2 通过元素自身实现比较规则

通过实现Comparable接口中的CompareTo方法。

例子：创建Users类，利用自身比较规则进行排序，其中年龄以小到大排序，年龄相等时，以按名字在字典中排序。

public class Users implements Comparable<Users>{

    private String uname;
    private int uage;

    public Users() {
    }

    public Users(String uname, int uage) {
        this.uname = uname;
        this.uage = uage;
    }

    public String getUname() {
        return uname;
    }

    public void setUname(String uname) {
        this.uname = uname;
    }

    public int getUage() {
        return uage;
    }

    public void setUage(int uage) {
        this.uage = uage;
    }

    @Override
    public String toString() {
        return "Users{" +
                "uname='" + uname + '\'' +
                ", uage=" + uage +
                '}';
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;

        Users users = (Users) o;

        if (uage != users.uage) return false;
        return uname != null ? uname.equals(users.uname) : users.uname == null;
    }

    @Override
    public int hashCode() {
        int result = uname != null ? uname.hashCode() : 0;
        result = 31 * result + uage;
        return result;
    }

    //自身比较规则
    //返回1，大。返回0，小。返回0，相等。
    @Override
    public int compareTo(Users o) {

        //年龄以小到大排序
        if (this.uage > o.getUage()){
            return 1;
        }
        //年龄相等时，以按名字在字典中排序
        if (this.uage == o.getUage()){
            //String类的comPareTo方法
            return this.uname.compareTo(o.getUname());
        }

        return -1;
    }
}
//自身比较规则测试
Set<Users> set1 = new TreeSet<>();

Users u1 = new Users("张三",19);
Users u2 = new Users("琲木木",22);
Users u3 = new Users("林三",22);

set1.add(u1);
set1.add(u2);
set1.add(u3);

for (Users users : set1){
    System.out.println(users);
}

总结：

1、如果类没有比较规则，那就通过实现Comparable接口进行重写CompareTo方法。

2、如果比较的是String类或者Integer类，就直接调用相应的CompareTo方法。

2.4.2.3 通过外部比较器指定比较规则

首先单独创建一个比较器，要实现Comparator接口中的compare方法来定义比较规则。

在实例化TreeSet时，提前将比较器交给TreeSet来完成元素的排序处理。

例子：创建Student类，利用外部比较器StudentComparator类进行排序。

public class Student {
    private String sname;
    private int sage;

    public Student() {
    }

    public Student(String sname, int sage) {
        this.sname = sname;
        this.sage = sage;
    }

    public String getSname() {
        return sname;
    }

    public void setSname(String sname) {
        this.sname = sname;
    }

    public int getSage() {
        return sage;
    }

    public void setSage(int sage) {
        this.sage = sage;
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;

        Student student = (Student) o;

        if (sage != student.sage) return false;
        return sname != null ? sname.equals(student.sname) : student.sname == null;
    }

    @Override
    public int hashCode() {
        int result = sname != null ? sname.hashCode() : 0;
        result = 31 * result + sage;
        return result;
    }

    @Override
    public String toString() {
        return "Student{" +
                "sname='" + sname + '\'' +
                ", sage=" + sage +
                '}';
    }
}
//外部定义比较器
public class StudentComparator implements Comparator<Student> {
    @Override
    public int compare(Student o1, Student o2) {
        if (o1.getSage() > o2.getSage()){
            return 1;
        }
        if (o1.getSage() == o2.getSage()){
            return o1.getSname().compareTo(o2.getSname());
        }
        return -1;
    }
}
//外部定义比较器测试
StudentComparator comparator = new StudentComparator();
Set<Student> set2 = new TreeSet<>(comparator);

Student s1 = new Student("张三",18);
Student s2 = new Student("琲木木",22);
Student s3 = new Student("林三",22);

set2.add(s1);
set2.add(s2);
set2.add(s3);

for (Student student : set2){
    System.out.println(student);
}

  • 同HashSet一样，只针对Map中的Key，而不用考虑Value。

2.4.3 单例集合使用案例

需求：

产生1-10之间的随机数([1,10]闭区间)，将不重复的10个随机数放到容器中。

2.4.3.1 使用List容器类实现
public class ListTestDemo {
    public static void main(String[] args) {

        List<Integer> list = new ArrayList<>();
        while (true){
            //产生随机数
            int num = (int)(Math.random()*10+1);

            //去重，判断元素是否存在。
            if (! list.contains(num)){

                //不存在即不重复，就添加到list中。
                list.add(num);
            }

            //循环结束条件
            if (list.size() == 10){
                break;
            }
        }
        for (Integer i : list){
            System.out.println(i);
        }
        /*
        Random r1 = new Random();
        //for循环的循环条件无法满足“不重复的10个随机数”，所以我们考虑让这个随机数一直跑。
        for (int i=0;i < 10;i++){
            list.add(r1.nextInt(10) + 1);
        }
        System.out.println(list);
        */
    }
}

注意：当我们无法满足条件时，我们就需要转换思路。

例如：for循环的循环条件无法满足“不重复的10个随机数”，所以我们考虑让这个随机数一直跑（while循环）。

2.4.3.2 使用Set容器类实现
public class SetTestDemo {
    public static void main(String[] args) {
        Set<Integer> set = new HashSet<>();
        while (true){
            int num = (int)(Math.random()*10+1);
            
            set.add(num);

            if (set.size() == 10){
                break;
            }
        }
        for (Integer integer : set){
            System.out.println(integer);
        }
    }
}

注意：

1、由于Set集合特性无序且不重复，所以不用考虑元素是否包含的问题。

2、假排序的问题，是由于Integer的hashCode方法，返回的就是元素本身。在经过哈希算法取模后存放挂在数组的位置正好是按照索引存放，造成了这种假排序现象。
