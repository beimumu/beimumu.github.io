7.封装（续）

5.static关键字

  1. static可以修饰全局变量，用static修饰的变量称为静态变量，静态变量不属于任何对象，静态变量被当前类的所有对象共享，静态变量可以使用类名.变量名直接访问。

    public class Student {
    public String name;

    public static String sex;
}
-------------------------------------
public class TestStudent {
    public static void main(String[] args) {
        Student.sex = "男";

        Student s1 = new Student();
        s1.name = "张三";

        Student s2 = new Student();
        s2.name = "李四";
        System.out.println(Student.sex);
        System.out.println(s1.sex == s2.sex);
    }
}

    运行结果：

    从运行结果看：静态变量确实被本类的所有对象共享。

  2. 内存原理：

  3. static可以修饰方法，被static修饰的方法称之为静态方法。静态方法可以使用类名.方法名直接调用。

    Student.attendClass();

  4. static可以用来修饰代码块，被static修饰的代码块称之为静态代码块，静态代码块在当前类被加载进内存的时候就会执行，并且整个程序运行期间，先于main()方法之前执行，只执行一次。主要作用：给静态变量初始化。

    运行结果：

    可以看出：静态代码块优先于主方法（静态方法和构造方法），最先被执行。

    • 程序执行顺序：静态代码块→静态方法→普通代码块→构造方法

    由此也可以得出在静态代码块里可以访问静态成员以及静态方法，但不能访问实例成员。

    • 内存分配顺序：静态变量→静态方法→静态代码块→构造方法

  5. 静态方法内不能直接调用非静态的成员(实例属性、实例方法)。

    • 当为普通方法时，方法间可以相互调用：

    • 当被static修饰了之后，实例成员就不能被静态方法所调用。

      如图：出现报错(Non-static 方法 'exam()' cannot be referenced from a static context)。

      因为在JVM给静态方法分配内存时，此时的非静态变量和方法还没有出现。

  6. 静态方法、静态代码块内不能出现this。静态方法中，实例成员还没出现呢，哪来的this关键字。

类加载的机制：

JVM虚拟机，类加载器负责把Java程序编译后的字节码文件加载进内存，加载字节码文件进内存后，JVM就会首先给当前类中的静态成员去分配空间。创建对象的时候，才会给实例成员分配空间。反过来，实例方法可以访问静态成员。

内存分配的顺序：

静态变量 → 静态方法 → 静态代码块 → 实例变量 → 构造方法 → 实例方法

6.设计模式——单例模式

  1. 设计模式（Design pattern）：

    • 设计模式是一套被反复使用、多数人知晓的、经过分类编目的、代码设计经验的总结。
    • 使用设计模式是为了可重用代码、让代码更容易被他人理解、保证代码可靠性。
    • 由GOF四人帮编写23种常用的设计模式。

  2. 单例模式：

    1. 在整个程序生命周期内，某个类只有一个实例存在（即只能存在一个对象）。这个实例必须自行实例化（即自行创建对象），并且向外界提供这个实例（即外界只能访问，但不能修改）。

    2. 实现要点：

      • 构造方法私有化

        解析：为什么要构造方法私有化？因为在外界创建对象时。可以创建多个对象，而这些对象是通过调用Jvm所创建的构造方法，所以我们需要自行创建一个构造方法，这样JVM就不会自主创建了，同时将构造方法进行私有化，外界就不能创建对象了。换言之，杜绝外界创建多个对象。

      • 私有的静态变量持有本类的唯一对象

        解析：由于构造方法私有化，外界不能创建对象（所以我们就的自己创建对象，我们将自己创建的对象作为全局变量存在。）进而不能访问类中的全局变量。此时我们把刚才的全局变量加上static关键字后，外界就可以通过类名.变量访问到了。如果不将静态对象私有化，则外界就可以随意更改变量的值，若Phone.p = null时，创建的对象在堆内存中会因为没有变量指向而导致被JVM当作垃圾清理掉。换言之，杜绝外界随意修改对象的值。

      • 公有的静态方法用来向外提供本类的唯一对象

        解析：因为在类中的实例方法创建后，由于对象已经私有化，外界访问不到，所以我们需要将实例方法变成静态方法。这样外界就可以通过方法来创建对象。

        如下图所示：

        通过运行结果可以发现，创建的三个对象的内存地址是一样的，可以得出结论，外界创建的对象是同一个，即完成了类中只有一个实例。

  3. 单例模式代码

    public class Phone {
    
    //2、私有的静态变量持有本类的唯一对象
    private static Phone p = new Phone();

    //1、构造方法私有化
    private Phone(){

    }

    //3、公有的静态方法用来向外界提供本类的唯一对象
    public static Phone getPhone(){
        return p;
    }
}

    恶汉式和懒汉式的区别就在于创建对象位置不同，也就导致运行加载速度不同。恶汉式在方法执行前提前创建，懒汉式则是当你需要使用方法时才会创建对象。

    //懒汉式
public class Phone {
    
    private static Phone p;
    
    private Phone(){

    }

    public static Phone getPhone(){
        if(p == null){
            p = new Phone();
        }
        return p;
    }
}

8.继承

1.子类与父类的关系

  • is-a ：意味 “子类是一个父类的…”。
  • 父类更抽象，更普通；子类更具体，更特殊。例：车-汽车，自行车，电动车。

2.继承的概念

  • 基于已经存在的类，来构建新类，叫做继承。
  • 使用extends关键字来实现。被继承的类，称之为父类或超类，延伸出来的新类叫做子类。
  • 子类会继承父类中所有的属性和方法。
  • 每当有子类对象被创建的时候，都会先创建一个父类对象（创建的子类对象中，有父类对象存在）。
  • 每当执行子类代码时，也会先进行父类的构造方法，在进行子类的构造方法。

3.继承的模型图

4.关键字this和super的区别

this&super 区 别
this 本类的引用
super 子类对父类对象的引用
this() 可以调用本类其他的构造方法（构造方法的第一行）
super() 在子类中，可以调用父类的构造方法（子类构造方法的第一行）
  • 栈：stack。给方法的执行临时开辟空间，方法执行结束后，此内存区域也会被释放掉。除了存放引用变量，方法内的局部变量（声明的一些基本数据变量，变量值）也会存放在栈内存中。执行速度比较快。
  • 堆：heap。存放对象。执行速度比较慢。

5.例子：人类-学生类

//Person类
public class Person {
    String name; //姓名
    String sex; //性别
    String id; //身份证号

    public Person() {
        this("name","sex","id");
        System.out.println("person的无参构造方法");
    }

    public Person(String name, String sex, String id) {
        this.name = name;
        this.sex = sex;
        this.id = id;
        System.out.println("person的有参构造方法");
    }

    public void sleep(){
        System.out.println("人睡觉");
    }
}
-----------------------------------------------------------------
//Student类
public class Student extends Person {
    String grade_class; //年级班级


    public Student() {

    }

    public Student(String name, String sex, String id, String grade_class) {
        super(name, sex, id);
        this.grade_class = grade_class;
        System.out.println("student的有参构造方法");
    }

    @Override
    public String toString() {
        return "Student{" +
                "grade_class='" + grade_class + '\'' +
                ", name='" + name + '\'' +
                ", sex='" + sex + '\'' +
                ", id='" + id + '\'' +
                '}';
    }
}
----------------------------------------------------------------------------
//TestStudent类
public class TestStudent {
    public static void main(String[] args) {
        Student student = new Student("张三","男","1234","三年二班");
        System.out.println(student);
    }
}

运行结果也证实执行子类构造方法前会先执行父类的构造方法。

6.方法的重写（Override）

  1. 概念

    方法的重写是指子类对从父类继承过来的方法，可以进行重写。

  2. 特点

    • 重写方法和被重写方法必须具有相同的返回值类型、方法名、参数列表。
    • 重写方法不能使用比被重写方法更高的访问权限。
    • 父类中的私有方法不能被重写。

  3. @Override：注解，标志着当前的方法是一个重写过的方法

    @Override
public void sleep(){
        System.out.println("学生睡觉");
}

7.final关键字

  • final关键字的特点

    • final可以修饰变量、方法、类。

    • 被final修饰过的变量称为常量，所以必须赋初值（initialization）。可以调用此常量，但不能修改。

    • 常量一般和static关键字配合使用（静态常量名必须大写）

      final static int A = 10;//静态常量

    • 被final修饰过的方法称为最终的方法，最终的方法不能被重写。

    • 被final修饰过的类称为最终的类，最终的类不能被继承。

8.Object类

  • Object类：在Java中，是所有类的祖先。所有类都直接或间接继承自Java.lang,Object类。

  • 常用方法：

    • toString()
    • equals()
    • hashcode()
    • clone()
    • getClass()
    • finalize()

  • equals()

    • Object类

      1.在Object类中的equals()方法默认是比较两个对象是否为同一个对象。

      public boolean equals(Object obj) {
    //this为调用此方法的对象
    //obj为被比较的对象
    return (this == obj);
}

      2.例子：通过重写equals()方法来判断是否为同一对象

      //Dog类
public class Dog {
    String color;
    String dogID;

    //判断d1和d2两个对象的属性值是否相等，如果相等则认为是同一个对象。
    @Override
    public boolean equals(Object obj) {
        //this为调用此方法的对象(比较的对象)。
        //obj为被比较的对象。但因数据类型不同，所以需要将“Object类型” 强制转换为 “比较的对象的数据类型”。
        Dog dog = (Dog)obj;

        if (this.color == dog.color && this.dogID == dog.dogID){
            return true;
        }
        return false;
    }
}
//TestDog类
public class TestDog {
    public static void main(String[] args) {
        Dog d1 = new Dog();
        d1.color = "blue";
        d1.dogID = "010";

        Dog d2 = new Dog();
        d2.color = "blue";
        d2.dogID = "010";

        if(d1.equals(d2)){
            System.out.println("信息比对正确，这是大黄");
        }else{
            System.out.println("信息比对错误，这不是大黄");
        }

}

    • String类

      在String类中的equals()方法

          public boolean equals(Object anObject) {
        //this 表示调用该方法的对象
        //anObject 表示被比较的对象
        if (this == anObject) {
            return true;
        }
        // instanceof关键字可以判断左边对象是否是右边类或者子类的一个实例
        //需要用到对象的强制类型转换时，需要使用instanceof进行判断。
        if (anObject instanceof String) {
            String anotherString = (String)anObject;
            //value 实际上就是String类中的一个字符型常量数组char value[];
            //说到底，string最后还是字符数组进行存储
            
            //调用该方法的对象的值的数组长度
            int n = value.length;
            //被比较的对象的数组长度
            //pan'duan
            if (n == anotherString.value.length) {
                char v1[] = value;
                char v2[] = anotherString.value;
                int i = 0;
                //长度自减，不等于0，执行while循环
                while (n-- != 0) {
                    //分别对应两个对象中字符是否相等
                    if (v1[i] != v2[i])
                        return false;
                    i++;
                }
                return true;
            }
        }    
        return false;
}

      拓展：
      instanceof关键字：

      1. 首先返回值类型是布尔值
      2. 比较过程中左边一般是对象，右边是类
      3. 作用是来判断左边是否属于右边的类或者是该类的子类创建的一个实例对象
      4. 需要用到对象的强制类型转换时，需要使用instanceof进行判断。
